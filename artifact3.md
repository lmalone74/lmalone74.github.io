# Artifact Three - Grazioso Salvare Database
**Database Enhancement**

A full-stack dashboard application enhanced with secure database integration, modular architecture, and interactive data visualization to support efficient data management.
>
### [View Original Code (GitHub)](https://github.com/lmalone74/Artifact-Three-Grazioso-Salvare/tree/Original-Code) | [View Enhanced Code (GitHub)](https://github.com/lmalone74/Artifact-Three-Grazioso-Salvare/tree/Enhanced-Code)
>

For my third artifact, I selected the Grazioso Salvare Animal Shelter Dashboard from CS 340 (2025). It is a Python-based application that connects to a MongoDB database and allows users to view, filter, and analyze animal shelter data through an interactive table, chart, and map. The original version was created in a Jupyter Notebook as part of a database assignment.

I chose this artifact because it demonstrates my ability to work with databases, integrate backend data with a user interface, and build a data-driven application. It highlights my use of CRUD operations, data handling with pandas, and interactive visualization using Dash. The enhancement focused on improving structure and security by refactoring the application into separate modules, removing hardcoded credentials in favor of environment variables, adding a login system to restrict access, and implementing input validation in the CRUD layer to ensure data integrity and prevent invalid input.

Through this process, I learned the importance of modular design and securing application data. A key challenge was restructuring the notebook into a multi-file application while maintaining functionality, which required careful testing. I also encountered challenges related to setting up the development environment and dependencies, as well as validating functionality during the transition from the notebook to a standalone application. I also had to ensure that database interactions remained accurate and secure after introducing authentication and validation mechanisms. Feedback from earlier coursework influenced my decision to improve code organization and eliminate hardcoded values. These changes improved the reliability, maintainability, and security of the application.

This enhancement strongly supports Outcome 5 by demonstrating a security-focused approach to application design. The use of environment variables prevents sensitive credentials from being exposed, while the addition of authentication and input validation helps protect against unauthorized access and improper data handling. It also supports Outcome 4 through the use of modern tools and techniques, including Python, MongoDB, and modular application design, to build a scalable and maintainable solution.
