# Artifact Two - Airgead Banking
**Algorithms and Data Structure Enhancement**

A C++ financial application enhanced to support multiple investment scenarios, comparative analysis, and structured data output for improved decision-making.
>
### [View Original Code (GitHub)](https://github.com/lmalone74/Artifact-Two-Airgead-Banking/tree/Original-Code) | [View Enhanced Code (GitHub)](https://github.com/lmalone74/Artifact-Two-Airgead-Banking/tree/Enhanced-Code)
>

For the second artifact, I selected the Airgead Banking program from CS 210 (2024). It simulates investment growth using compound interest based on user inputs and generates reports with and without monthly deposits. The program demonstrates algorithmic logic and uses vectors to store yearly balances and interest, modeling financial growth through structured data and object-oriented programming in C++.

I chose this artifact because it highlights my ability to apply algorithms and data structures to solve a real-world problem. The program demonstrates my understanding of compound interest calculations, iterative processing, and the use of vectors to manage and store data over time. In its original form, the program was limited to a single scenario and had a basic structure. The enhancement introduced support for multiple investment scenarios using a vector-based approach and a dedicated class for managing scenario data. A summary report was added to improve sorting and comparison of results, and the application structure was refactored by separating functionality into classes and introducing a menu system. These changes improved usability, organization, and scalability of the program.

Through the process of enhancing this artifact, I strengthened my understanding of modular design and the importance of separating logic from control flow. One of the main challenges was restructuring the program without breaking existing functionality while ensuring calculations remained accurate across multiple scenarios. This required careful testing and validation of results. Feedback from earlier coursework influenced my decision to improve organization and make the program more user-friendly. The final version of the artifact is more flexible, easier to use, and better structured for future expansion.

This enhancement supports decision-making through comparison of multiple scenarios, improves data presentation with exportable summaries, and demonstrates structured data handling and sorting. It aligns with Outcome 1, through designing and improving a computing solution, and Outcome 2, through the use of data structures and algorithms to manage and analyze financial data. It also supports Outcome 4, as the enhancements focused on improving usability, organization, and maintainability, reflecting professional software development practices. While more advanced features such as a graphical interface were not implemented, the program demonstrates a strong foundation in algorithmic problem-solving and program design.
