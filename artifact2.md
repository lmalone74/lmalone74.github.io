# Artifact Two - Airgead Banking
**Algorithms and Data Structure Enhancement**

For the second artifact, I selected the Airgead Banking program from CS 210 (2024). It simulates investment growth using compound interest based on user inputs and generates reports with and without monthly deposits. The program demonstrates algorithmic logic and uses vectors to store yearly balances and interest, modeling financial growth through structured data and object-oriented programming in C++.

The program was enhanced to support multiple investment scenarios using a vector-based approach and a dedicated class for managing scenario data. A summary report was added to improve sorting and comparison of results. The application structure was refactored by separating functionality into classes and introducing a menu system, allowing users to run scenarios, save results, or exit cleanly. These changes improved usability, organization, and scalability of the program.

This enhancement supports decision-making through comparison of multiple scenarios, improves data presentation with exportable summaries, and demonstrates structured data handling and sorting.

It strengthened my understanding of modular design and separating logic from control flow. Challenges included restructuring without breaking functionality and ensuring accurate calculations across scenarios, which required careful testing.
