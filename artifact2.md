# Artifact Two - Airgead Banking
**Algorithms and Data Structure Enhancement**

A C++ financial application enhanced to support multiple investment scenarios, comparative analysis, and structured data output for improved decision-making.
>
### [View Original Code (GitHub)](https://github.com/lmalone74/Artifact-Two-Airgead-Banking/tree/Original-Code) | [View Enhanced Code (GitHub)](https://github.com/lmalone74/Artifact-Two-Airgead-Banking/tree/Enhanced-Code)
>

For the second artifact, I selected the Airgead Banking program from CS 210 (2024). It simulates investment growth using compound interest based on user inputs and generates reports with and without monthly deposits. The program demonstrates algorithmic logic and uses vectors to store yearly balances and interest, modeling financial growth through structured data and object-oriented programming in C++.

I chose this artifact because it highlights my ability to apply algorithms and data structures to solve a real-world problem. The program demonstrates my understanding of compound interest calculations, iterative processing, and the use of vectors to manage and store data over time. In its original form, the program was limited to a single scenario and had a basic structure. The enhancement introduced support for multiple investment scenarios using a vector-based approach and a dedicated class for managing scenario data. This required restructuring how data was stored and accessed, shifting from a single linear calculation to managing multiple independent data sets, each with its own lifecycle and output. A summary report was added to improve sorting and comparison of results, and the application structure was refactored by separating functionality into classes and introducing a menu system. These changes improved usability, program organization, and the ability to scale the application to support additional scenarios.

Through this process, I strengthened my understanding of modular design and how separating responsibilities across classes improves both maintainability and scalability, particularly when expanding a program beyond its original scope. One of the main challenges was restructuring the program without breaking existing functionality while maintaining accurate calculations across multiple scenarios, this required careful testing and validation of results. One challenge I encountered was ensuring that data remained consistent across multiple scenarios, particularly when restructuring the program to support comparison and sorting without introducing calculation errors. Feedback from earlier coursework influenced my decision to improve organization and make the program more user-friendly. The final version of the artifact is more flexible, easier to use, and better structured for future expansion.

This enhancement supports Outcome 1 by enabling more effective decision-making through the comparison of multiple investment scenarios. It supports Outcome 2 through the use of data structures and algorithms, including vector-based storage and sorting, to manage and analyze financial data. It also supports Outcome 3 by improving software organization through modular design and separation of functionality, resulting in a more maintainable and scalable application.
